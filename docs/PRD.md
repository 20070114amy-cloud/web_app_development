# 專案名稱：任務管理系統 (Task Management System)

## 1. 專案概述
- **背景與動機**：現代人生活與工作繁忙，常會忘記許多待辦事項。為了幫助提升生產力與時間管理，我們需要一個簡單而直觀的任務管理系統，來幫助記錄並追蹤日常工作。
- **目標用戶**：需要管理日常工作的個人用戶。
- **核心價值主張**：以最簡單、快速的方式，讓使用者能夠隨時記錄、查看及管理待辦任務，幫助清空大腦、減輕記憶負擔。

## 2. 功能需求
主要功能：
1. **新增任務**：作為一名個人用戶，我希望可以新增一筆待辦任務，以便將腦中需要做的事情記錄下來。
2. **顯示任務清單**：作為一名個人用戶，我希望可以一目了然地看到所有建立的任務，以便了解當前所有的工作全貌。
3. **標記任務完成**：作為一名個人用戶，我希望可以將已處理完畢的任務標記為完成，以便追蹤進度並確認當天的產出。
4. **依完成狀態篩選**：作為一名個人用戶，我希望可以篩選顯示「未完成」或「已完成」的任務，以便更專注地處理當下急需處理的任務。
5. **刪除任務**：作為一名個人用戶，我希望可以刪除不再需要或誤填的任務，以便保持任務清單的整潔。

## 3. 非功能需求
- **技術限制**：
  - 前端渲染：使用 **Jinja2** 進行 HTML 模板渲染。
  - 後端框架：使用 **Python + Flask** 建立 Web 服務。
  - 資料庫：使用 **SQLite** 進行輕量化的資料儲存。
- **效能與安全考量**：
  - 系統應確保在個人單機或輕量使用下，頁面載入速度迅速且資源操作反應即時。
  - 透過 Jinja2 的自動轉義功能，防範基本的 XSS (跨站腳本) 等注入攻擊，確保輸入資料防護。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**：
  - 新增任務
  - 顯示任務清單
  - 標記任務完成
- **Should Have (應該有)**：
  - 刪除任務
  - 依完成狀態篩選
- **Nice to Have (可以有)**：
  - 編輯任務內容
  - 設定任務優先級 (高/中/低)
  - 任務到期日與提醒
